双指针

双指针主要用于遍历数组，两个指针指向不同的元素，从而协同完成任务。

1. 有序数组的 Two Sum

题目描述

在有序数组中找出两个数，使它们的和为 target。

题解

使用双指针，一个指针指向值较小的元素，一个指针指向值较大的元素。指向较小元素的指针从头向尾遍历，指向较大元素的指针从尾向头遍历。

  • 利用两个指针分别从前向后以及从后向前遍历
    • 如果两个指针指向元素的和 sum == target，那么得到要求的结果；
    • 如果 sum > target，移动较大的元素，使 sum 变小一些；
    • 如果 sum < target，移动较小的元素，使 sum 变大一些。

  • hash表的查询效率为 $O(1)$ ，故如果内层循环的查找使用hash表来实现，则总复杂度即为 $O(n)$ 。

    • 在hash表中查询是否存在对应的j，若找到，则直接return。

2. 两数平方和

题目描述

判断一个非负整数是否为两个整数的平方和。

题解

解法1

利用C++的运算机制，主要是判断对每个 $a$ ， $c-a^2$ 能否开方。
需要注意的是

  • 防止爆0：
    • c是可以为0的，而此种写法必须特判；
  • 防止爆int：
    • 若c = 2147482647，则 $a^2$ 一定会爆int，因而需要使用long long替换；
    • 但随之而来的就是，内存使用将会增加；

解法2 - 双指针

同两数之和的题，使用双指针的思想会更加简洁，方便。

实际上两种解法的空间消耗相差无几。

4. 回文字符串

题目描述

给定一个非空字符串 s，最多删除一个字符。判断是否能成为回文字符串。

题解

由于最多只能删除一个字符，因此，当双指针指向的元素不同时，只需要判断下列两个子串是否满足回文串即可：

  • (i+1, j)
  • (i, j-1)

注意，这里使用了inline函数，使执行用时从76ms提升到了72ms，击败91.07%的用户。

一开始的代码为

s[i+1]==j 或者 s[i]==s[j-1] ，但忽略了这两者是可能同时成立的，比如这个例子

6. 判断链表是否存在环

题目描述

给定一个链表，判断链表中是否有环。

题解

解法1 - 快慢指针法

通过使用具有 不同速度 的快、慢两个指针遍历链表，空间复杂度可以被降低至 O(1)。慢指针每次移动一步，而快指针每次移动两步。

  • 如果链表中不存在环，最终快指针将会最先到达尾部，此时我们可以返回 false。
  • 如果链表中存在环，快指针一定会追上慢指针

解法2 - 标记法
